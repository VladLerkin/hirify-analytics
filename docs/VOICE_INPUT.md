# Голосовой ввод в Hirify Analytics

## Описание

Функция голосового ввода позволяет добавлять родственников в генеалогическое древо с помощью голосовых команд. Система автоматически распознаёт речь, обрабатывает её через AI и создаёт структурированные данные о людях и связях между ними.

## Как использовать

### Android
1. Откройте меню (три точки в правом верхнем углу)
2. Выберите "Voice Input 🎤"
3. Разрешите доступ к микрофону при первом использовании
4. Произнесите фразу с описанием родственников
5. Дождитесь завершения обработки

### Desktop
Голосовой ввод не поддерживается на Desktop платформе в текущей версии.

### iOS
Голосовой ввод не поддерживается на iOS платформе в текущей версии.

## Примеры голосовых команд

### Простой пример
```
Добавь Петра Ивановича Петрова и его жену Клавдию Ивановну
```

### Пример с детьми
```
Добавь Петра Ивановича Петрова и его жену Клавдию Ивановну и их детей Петра и Февронию
```

### Расширенный пример
```
Добавь Ивана Петровича родился в 1950 году, его жену Марию Сергеевну родилась в 1952 году, и их троих детей: Александра родился в 1975, Елену родилась в 1977, и Дмитрия родился в 1980
```

## Как это работает

1. **Распознавание речи**: На Android используется встроенный SpeechRecognizer API для преобразования голоса в текст (русский язык). Поддерживаются следующие провайдеры транскрипции:
   - **OpenAI Whisper** - высокая точность, поддержка многих языков
   - **Google Speech-to-Text** - быстрая обработка, хорошая точность
   - **Yandex SpeechKit** - отличная поддержка русского языка и языков СНГ
2. **Обработка через AI**: Распознанный текст отправляется в настроенный LLM (OpenAI, Anthropic, Ollama или Custom)
3. **Извлечение структуры**: AI анализирует текст и извлекает:
   - Имена и фамилии людей
   - Пол (если указан или можно определить)
   - Даты рождения и смерти (если указаны)
   - Родственные связи (супруги, родители, дети)
4. **Создание дерева**: Система автоматически создаёт записи в базе данных и связи между ними
5. **Отображение**: Новые люди появляются на канвасе с автоматическим позиционированием

## Требования

### Android
- Android 5.0 (Lollipop) или новее
- Разрешение RECORD_AUDIO
- Подключение к интернету (для распознавания речи через OpenAI Whisper, Google Speech-to-Text или Yandex SpeechKit)
- Настроенный AI сервис (через меню "AI Settings...")

### Настройка AI
Перед использованием голосового ввода необходимо настроить AI:
1. Откройте меню → "AI Settings..."
2. Выберите провайдера (OpenAI, Anthropic, Ollama, Custom)
3. Введите API ключ (для OpenAI/Anthropic) или URL (для Ollama/Custom)
4. Выберите модель
5. Сохраните настройки

## Ограничения

- Голосовой ввод работает только на **Android** устройствах
- Требуется активное подключение к интернету для распознавания речи
- Качество распознавания зависит от:
  - Чёткости произношения
  - Уровня окружающего шума
  - Качества микрофона
- AI может неправильно интерпретировать сложные родственные связи
- Рекомендуется проверять результат после добавления

## Устранение неполадок

### "Голосовой ввод недоступен на этой платформе"
- Вы используете Desktop или iOS версию приложения
- Голосовой ввод доступен только на Android

### "Недостаточно разрешений"
- Предоставьте разрешение на использование микрофона в настройках Android
- Настройки → Приложения → Hirify Analytics → Разрешения → Микрофон

### ⚠️ СПЕЦИАЛЬНО ДЛЯ УСТРОЙСТВ XIAOMI/MIUI
MIUI имеет дополнительную многоуровневую систему безопасности и разрешений, которая может блокировать голосовой ввод даже если разрешение RECORD_AUDIO предоставлено в стандартных настройках Android.

**Симптомы проблемы на MIUI:**
- Разрешение микрофона предоставлено, но появляется ошибка "Недостаточно разрешений"
- В логах видно: "RECORD_AUDIO permission granted", но затем "ERROR_INSUFFICIENT_PERMISSIONS"
- В системных логах: "RTMode: pkgName: com.family.tree.android has no permission"

**Решение для MIUI (выполните ВСЕ шаги по порядку):**

1. **Основные разрешения приложения:**
   - Настройки → Приложения → Все приложения → Hirify Analytics
   - Разрешения → Микрофон → Разрешить

2. **Глобальные разрешения MIUI:**
   - Настройки → Приложения → Разрешения → Микрофон
   - Убедитесь что Hirify Analytics в списке разрешённых приложений

3. **Система безопасности MIUI:**
   - Настройки → Защита (Security) → Разрешения → Микрофон
   - Найдите Hirify Analytics и включите разрешение
   - Если есть опция "Разрешить всегда" - выберите её

4. **Оптимизация батареи:**
   - Настройки → Батарея → Энергосбережение → Выберите приложение
   - Найдите Hirify Analytics → Без ограничений
   - Это важно, так как MIUI может блокировать сенсоры при оптимизации

5. **Автозапуск (опционально):**
   - Настройки → Приложения → Автозапуск
   - Включите автозапуск для Hirify Analytics
   - Это может помочь с фоновыми сервисами

6. **Перезапуск:**
   - После изменения всех настроек полностью закройте приложение
   - Очистите его из списка недавних приложений
   - Запустите заново

**Если проблема сохраняется:**
- Попробуйте перезагрузить устройство
- Проверьте, работает ли голосовой ввод в других приложениях (Google, Telegram)
- Обновите MIUI до последней версии
- В крайнем случае: сброс настроек приложения (Настройки → Приложения → Hirify Analytics → Очистить данные)

### "Речь не распознана"
- Говорите громче и чётче
- Уменьшите уровень фонового шума
- Проверьте работу микрофона в других приложениях

### "Ошибка обработки текста через AI"
- Проверьте настройки AI в меню "AI Settings..."
- Убедитесь, что API ключ действителен
- Проверьте подключение к интернету
- Попробуйте другого AI провайдера

## Технические детали

### Архитектура
```
VoiceRecorder (Android) 
    ↓ (распознанный текст)
VoiceInputProcessor
    ↓ (текст)
AiTextImporter
    ↓ (запрос к AI)
Transcription API (OpenAI Whisper / Google Speech / Yandex SpeechKit)
    ↓ (транскрибированный текст)
LLM API (OpenAI/Anthropic/Ollama/Custom)
    ↓ (JSON)
ProjectData (люди + связи)
    ↓
Canvas (отображение)
```

- `/core/src/commonMain/kotlin/com/family/tree/core/platform/VoiceRecorder.kt` - Интерфейс

- `/core/src/desktopMain/kotlin/com/family/tree/core/platform/VoiceRecorder.desktop.kt` - Desktop заглушка
- `/core/src/iosMain/kotlin/com/family/tree/core/platform/VoiceRecorder.ios.kt` - iOS заглушка
